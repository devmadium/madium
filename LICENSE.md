## 🔧 Common Issues & Fixes

**❌ "Injection failed" or no confirmation after clicking Inject**
> Run `Madium.exe` as Administrator. If the issue persists, check your antivirus quarantine folder — the executor DLL is likely being blocked. Restore the file, whitelist the entire Madium folder, and retry.

**❌ Roblox crashes immediately after injection**
> Your Madium build is most likely outdated following a recent Roblox client update. Download the latest release from this repository and perform a clean installation by deleting your existing Madium folder before extracting the new build.

**❌ Script executes but has no visible effect in-game**
> This is a script compatibility issue, not an executor issue. The script you are running may be outdated or written for a different game version. Test a Script Hub entry to confirm Madium itself is working correctly, then source an updated version of your script.

**❌ GUI opens off-screen or is completely invisible**
> Delete `config.json` from your Madium installation folder and restart the executor. This resets all window positions and display settings to their factory defaults.

**❌ Auto-Attach does not trigger when Roblox launches**
> Ensure Roblox is fully closed before opening Madium. Launch Madium first with Auto-Attach enabled in the settings panel, then launch Roblox. Auto-Attach cannot retroactively connect to an already-running game process.

**❌ Script Hub displays "Failed to load" or appears blank**
> This is a network connectivity issue. Check your internet connection and ensure your firewall is not blocking Madium's outbound HTTPS traffic. Temporarily disabling a VPN resolves this in the majority of cases.

**❌ Executor flagged by Windows Defender as a trojan or virus**
> This is a false positive caused by the DLL injection mechanism. Right-click `Madium.exe` → Properties → Unblock, then add the Madium folder to Windows Defender's exclusion list via Settings → Windows Security → Virus & threat protection → Exclusions.
